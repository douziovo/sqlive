    clearDatabase() no longer calls `PRAGMA foreign_keys = OFF` or `PRAGMA foreign_keys = ON`. DROP TABLE order follows leaf-to-root FK dependency order. Cycle detection logs warning but still drops all tables.
  </acceptance_criteria>
  <action>
    Rewrite `clearDatabase(JdbcTemplate jdbc)` (lines 103-126) as follows:

    1. **Query all objects from sqlite_master** (keep existing query for views, triggers, tables — it already groups by type).

    2. **Build FK dependency graph for tables:**
       - Call `metadataExtractor.extractForeignKeys(jdbc)` to get `List<ForeignKeyInfo>`.
       - Build adjacency list: `Map<String, List<String>>` where key = table name, value = list of tables that this table references (FK "to" table). Each ForeignKeyInfo has `fromTable` -> `toTable`.
       - Collect all table names from the sqlite_master query result (tables only, not views/triggers).

    3. **Topological sort (Kahn's algorithm, O(V+E)):**
       ```
       inDegree = map of tableName -> count of tables referencing it
       queue = all tables with inDegree 0
       sorted = []
       while queue not empty:
         node = queue.poll()
         sorted.add(node)
         for each dependent of node:
           inDegree[dep]--
           if inDegree[dep] == 0: queue.add(dep)
       if sorted.size() < total tables → cycle detected: log.warn("FK cycle detected, dropping tables in arbitrary order") → use remaining tables unsorted
       ```

    4. **Drop in leaf-to-root order:** sorted as computed is already leaf-first (referencing tables before referenced tables). Tables with no FK dependents (leaves) drop first. Tables that others depend on drop last. No reversal needed — Kahn's natural sorted order IS the correct DROP order.

    5. **Drop sequence:**
       - Drop views first (same as before, they don't participate in FK)
       - Drop triggers second (same as before)
       - Drop tables in topological order (leaf-first, as computed by Kahn's algorithm)
       - All drops happen inside a single `jdbc.execute((Connection con) -> { ... })` block (keep the same transactional wrapper)

    6. **Remove PRAGMA foreign_keys = OFF/ON calls** (lines 109 and 122). The entire point is that correct DROP ordering makes FK toggling unnecessary (D-08).

    7. **Handle edge cases:**
       - Empty database (no tables) — no DROP statements, no PRAGMA calls
       - Tables not participating in FK (no in-degree, no out-degree) — drop in any order among themselves
       - Circular FK — log.warn with message including table names involved in cycle, then drop remaining tables without ordering

    8. **Modify imports:** Add `java.util.*` if not already present (needed for Map, HashMap, ArrayList, LinkedList, Queue). The file already imports `java.util.ArrayList` and `java.util.List`. Replace with `java.util.*`.
  </action>
