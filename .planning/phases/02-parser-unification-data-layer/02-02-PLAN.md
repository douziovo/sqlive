<objective>
Unify SQL parsing between frontend and backend by having the backend expose canonical statement character offsets that the frontend uses instead of its own parser.

Purpose: CORE-01 bridges the parser gap. The backend SqlParser correctly handles BEGIN...END blocks and CASE...END depth tracking — the frontend extractSqlStatements doesn't. By consuming backend canonicalStatements, inline cell edits (updateRow/deleteRow/dropTableUI) use the same statement boundaries the backend used for execution, eliminating silent edit failures on complex scripts.
Output: Modified SqlParser.java, SqlResponse.java, SqlExecutionService.java, DatabaseTypes.ts, ApiTypes.ts, useSqlEngine.ts, useBidirectionalSync.ts
</objective>
