<objective>
Preserve ghost row input state on failed insert, allowing user correction and retry (FRONT-03).

Purpose: When a user fills the ghost row and submits a new row insertion, if the SQL execution fails (constraint violation, FK violation, timeout), the ghost row is currently cleared -- the user loses all typed input. This fix keeps the ghost row visible with its values intact on failure, adds visual feedback (red-tinted background, dashed border, error hint text), and lets the user edit values and retry. Success is detected by watching the target table's row count increase.

Output: Modified TableSection.vue (ghost row lifecycle logic + failure UI)
</objective>

        if (ghostPending.value && newLen > previousRowCount) {

        ghostPending.value = false
        ghostFailed.value = true
      }
