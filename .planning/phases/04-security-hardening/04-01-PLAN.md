<objective>
Add ATTACH DATABASE and PRAGMA blocking to SqlExecutionService.

Purpose: Prevent SQL injection attack vectors that grant server filesystem access (ATTACH DATABASE) or leak server information (PRAGMAs). Blocking happens at the statement validation layer, before SQLite executes the statement, and does not rely on SQLite's authorizer callback. Even though each request resets the database (per D-04, a single request still has filesystem access risk via ATTACH), the pre-execution check eliminates this entirely.

Output: Modified `SqlExecutionService.java` with `isBlockedStatement()` private method and injection point in the per-statement execution loop.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User SQL input → SqlExecutionService.execute() | Untrusted SQL text crosses into the statement execution loop. ATTACH DATABASE and PRAGMA statements are vectors for server filesystem access and information disclosure. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering / Information Disclosure | SqlExecutionService.execute() per-statement loop | mitigate | Pre-execution regex check using `ATTACH_PATTERN` — blocks `ATTACH DATABASE 'path'` and shorthand `ATTACH 'path'` before SQLite executes. Returns clear error with line number per D-09. ASVS L1: Input Validation. |
| T-04-02 | Information Disclosure | SqlExecutionService.execute() per-statement loop | mitigate | Pre-execution regex check using `PRAGMA_PATTERN` — blocks ALL user-submitted PRAGMA statements (not a blacklist approach per D-02 rationale). Returns clear error with line number per D-10. ASVS L1: Output Encoding / Information Disclosure. |
| T-04-03 | Tampering | Backend internal PRAGMA calls | accept | Backend's own `PRAGMA foreign_keys = OFF`, `PRAGMA index_info(...)`, `PRAGMA foreign_key_list(...)` execute outside the per-statement loop through independent `jdbc.execute()`/`jdbc.query()` calls. Not affected by the blocking check (D-03). |
| T-04-SC | Tampering | Build dependencies | accept | No new dependencies introduced. Pure Java change using existing `java.util.regex.Pattern` (available via existing `import java.util.*`). |
</threat_model>
